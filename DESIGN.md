# DESIGN.md — clawhub.md

Generated by /plan-design-review on 2026-03-24.
All implementation decisions calibrate against this document.

## Aesthetic Direction

**Code-native. Monospace. Dark mode first.**

clawhub.md is a developer tool. The aesthetic communicates "this is native to your environment"
— not a SaaS marketing page. It should feel like opening a well-designed terminal app.

Reference: charm.sh, mango.sh (aesthetic) + pkg.go.dev (information density)
Anti-reference: Vercel-clone SaaS, purple-gradient AI tools, card-grid registries

## Color System

```css
:root {
  --bg-primary:    #0d1117;  /* GitHub dark — main page background */
  --bg-secondary:  #161b22;  /* Code blocks, sidebar */
  --bg-install:    #1f2937;  /* Install command box */
  --bg-elevated:   #21262d;  /* Hover states, skill list items */

  --text-primary:  #e6edf3;  /* Body text */
  --text-secondary:#8b949e;  /* Author, metadata, secondary labels */
  --text-muted:    #484f58;  /* Dividers, placeholder text */

  --accent:        #58a6ff;  /* Install box border, links, copy button hover */
  --accent-green:  #3fb950;  /* "Copied!" confirmation */
  --accent-yellow: #d29922;  /* "Cached X hours ago" stale badge */
  --accent-red:    #f85149;  /* Error states */

  --border:        #30363d;  /* Dividers, box borders */
  --install-border:#58a6ff;  /* Left border on install command box */
}
```

**No other colors.** If a design decision requires a color not in this list, the decision is wrong.

## Typography

**Primary typeface: JetBrains Mono** (monospace — body, headings, all text)
- Source: https://www.jetbrains.com/legalforms/fonts/ (free, OFL license)
- Fallback: `'Courier New', monospace`

**No sans-serif typeface.** The monospace aesthetic is intentional and consistent.
The install command box feels native — not like a UI element pasted on top of a different design.

```css
:root {
  --font-mono: 'JetBrains Mono', 'Courier New', monospace;
  --font-size-xs:   11px;  /* metadata, captions */

  --font-size-lg:   18px;  /* skill description */
  --font-size-h2:   20px;  /* section headings */

}
```

## Install Command Box

The most critical UI element. Spec is non-negotiable:

```css
.install-box {
  background: var(--bg-install);         /* #1f2937 */
  border: 1px solid var(--border);       /* #30363d */
  border-left: 3px solid var(--accent);  /* #58a6ff — the one accent border allowed */
  border-radius: 6px;
  padding: 16px;
  font-family: var(--font-mono);
  font-size: 14px;
  display: flex;
  align-items: center;
  justify-content: space-between;
  overflow-x: auto;                      /* long commands scroll, don't wrap */
  -webkit-overflow-scrolling: touch;     /* mobile smooth scroll */
}
```

**INSTALL label:** Small caps, `#8b949e`, 11px. Always present.
**Copy button states:**
- DEFAULT: "Copy install command" — `#8b949e` text, no background
- HOVER: `#58a6ff` text
- COPYING: spinner 100ms
- SUCCESS: "Copied!" — `#3fb950`, 2 seconds, then reverts
- FAIL: command renders in `<input select-all>` for manual copy

**Copied string:** NEVER includes `$`, `%`, or `>` shell prompt characters.
**Long commands:** Horizontal scroll within the box. No wrapping.

## Skill List (homepage + browse)

Dense list, NOT cards. Each row:
```
[author/skill-name]    [description, max 60 chars, #8b949e]    [date added]
```
- Row height: 40px
- Hover: background → `#21262d`
- No icons, no badges, no decorative elements
- Full row is clickable

## Syntax Highlighting

**Page chrome: always dark.** No light mode toggle. Opinionated and consistent.

**Syntax highlighting within rendered markdown:** System-preference aware.
- `prefers-color-scheme: dark` → GitHub Dark (Highlight.js `github-dark` theme)
- `prefers-color-scheme: light` → GitHub Light (markdown content only; page stays dark)

Reason: GitHub themes are familiar to every developer. Zero "I picked a random theme" energy.

## Error & Empty States

### 404 — Skill Not Found (/r/ path)
```
clawhub.md
─────────────────────────────────────────────────────────

Skill not found

github.com/user/repo/path/skill.md
This skill may have moved or been deleted.

  [Search clawhub.md]

Browse curated skills →
```

### 429 / Rate Limited (stale content served)
The page renders normally with a banner:
```
┌─────────────────────────────────────────────────────┐
│  ⚠  Serving cached version · Last updated 2h ago   │
│     Add ?refresh=1 to force a fresh fetch           │
└─────────────────────────────────────────────────────┘
```
Banner color: `--accent-yellow` (#d29922), subtle, not alarming.

### No Install Command Found
Install box renders with degraded state:
```
┌─ NO INSTALL COMMAND DETECTED ──────────────────────┐
│  See full usage below ↓                             │
└─────────────────────────────────────────────────────┘
```
Box uses `--bg-secondary` background, no blue border. Full first code block shown below.

### Astro 404 (non-existent curated skill)
Same dark theme. Simple:
```
404 — Page not found
clawhub.md/author/skill is not in the curated registry.

Browse curated skills →   Or search any GitHub skill via /r/...
```

## Navigation

**Minimal.** Three elements only:
```
clawhub.md    [Browse]    [/r/ — any GitHub skill]
```
- No hamburger menu on mobile — nav collapses to just `clawhub.md` logo + icon row
- Nav height: 48px
- No megamenu, no dropdown, no secondary nav

## Responsive

**Mobile-first.** Max-width 800px is the desktop constraint. Below 640px:
- Install box: full width, horizontal scroll for long commands, font-size 12px
- Skill list: description truncated to 40 chars (vs 60)
- Nav: logo + two icons (Browse 🔍, /r/ #)
- Touch targets: minimum 44px height

**Keyboard navigation:**
- Tab stops: nav links → install copy button → skill list items
- Enter on copy button: triggers copy
- Focus ring: `outline: 2px solid var(--accent)`, `outline-offset: 2px`

## Anti-patterns (never do these)

1. Purple/gradient backgrounds
2. Card grid for skill browsing (use dense list)
3. Icons in colored circles
4. Centered text alignment on list items
5. Uniform bubbly border-radius on all elements
6. Decorative blobs or wavy dividers
7. Emoji as design elements in UI chrome
8. "Welcome to clawhub.md" copy
9. Badges: ⭐ Featured, 🔥 Popular, ✨ New
10. Generic hero section with tagline

## What the Install Box Communicates

The install box is the entire product in one UI element. Every design decision for it is optimized around one interaction: developer sees it → reads command → copies → runs. Under 10 seconds. The box must:
- Be the first thing that draws the eye (position + accent color)
- Answer "what do I run?" immediately (command text)
- Answer "can I trust this?" (source link below)
- Confirm the action worked (copied! feedback)
- Never require scrolling to see
